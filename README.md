# Streaming Analytics
A data analysis project for a fictional streaming platform, aimed at generating insights to support business decisions, such as subscription plan revenue, user distribution by age group, favorite genres, churn rate, and more.

## How to Run
1. Install Python
2. Install and run MongoDB locally
3. Create a .env file in the project root based on .env.example
4. Install the dependencies:

```
pip install -r requirements.txt
```
5. Run the pipeline:
```
python main.py
```
The generated CSV files will be available in the reports/ folder.


## Data Structure
Each generated user is stored in MongoDB with the following schema:

| Field | Type | Description |
|---|---|---|
| `_id` | ObjectId | Auto-generated by MongoDB |
| `name` | string | Full name |
| `email` | string | Email address |
| `age` | integer | Age between 18 and 80 |
| `country` | string | Country of origin |
| `plan` | string | `STANDARD_WITH_ADS`, `STANDARD` or `PREMIUM` |
| `monthly_fee` | float | `7.99`, `17.99` or `24.99` |
| `subscription_date` | string | Date of subscription (last 30 days) |
| `watch_hours_per_month` | integer | Watch hours between 10 and 70 |
| `is_active` | boolean | `true` for active users, `false` for churned |
| `cancel_date` | string \| null | Cancellation date (only for churned users) |
| `favorite_genre` | string | One of 10 available genres |

> 10,000 users are generated per run, with approximately 20% churn rate.

# Streaming Analytics
Projeto de análise de dados de uma plataforma de streaming fictícia, com o objetivo de gerar insights que possam orientar decisões de negócio, como retorno por plano de assinatura, distribuição de usuários por faixa etária, gêneros favoritos, taxa de cancelamento e muito mais.

## Como rodar
1. Instale o Python
2. Instale e rode o MongoDB localmente
3. Crie um arquivo .env na raiz do projeto seguindo o .env.example
4. Instale as dependências:

```
pip install -r requirements.txt
```
5. Execute o pipeline:
```
python main.py
```
Os CSVs gerados ficam na pasta reports/.


## Estrutura dos Dados
Cada usuário gerado é armazenado no MongoDB com o seguinte esquema:

| Campo | Tipo | Descrição |
|---|---|---|
| `_id` | ObjectId | Gerado automaticamente pelo MongoDB |
| `name` | string | Nome completo |
| `email` | string | Endereço de e-mail |
| `age` | integer | Idade entre 18 e 80 anos |
| `country` | string | País de origem |
| `plan` | string | `STANDARD_WITH_ADS`, `STANDARD` ou `PREMIUM` |
| `monthly_fee` | float | `7.99`, `17.99` ou `24.99` |
| `subscription_date` | string | Data de assinatura (últimos 30 dias) |
| `watch_hours_per_month` | integer | Horas assistidas entre 10 e 70 |
| `is_active` | boolean | `true` para usuários ativos, `false` para cancelados |
| `cancel_date` | string \| null | Data de cancelamento (apenas para usuários churned) |
